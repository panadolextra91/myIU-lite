<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| contributor PR → protected branch | The merge gate is the boundary; un-gated merges let broken/unreviewed code in |
| admin user → bypass controls | An admin can merge past a red check unless bypass is disabled, defeating the gate |
| throwaway proof branch → tracked branches | The deliberately-failing test must never leak into a real branch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Broken code merged to a protected branch | mitigate | Branch protection requires the `ci` check; demonstrated by a blocked failing PR. Acceptance: `mergeStateStatus: BLOCKED` + red `ci` captured. (INFRA-07, criterion #4) |
| T-03-02 | Elevation of Privilege | Admin merges past a red check | mitigate | "Include administrators / no bypass" enabled in Task 1 (Pitfall 3). Acceptance: user confirms admins included. |
| T-03-03 | Denial of Service | Required check never reports (name mismatch) | mitigate | Required check name equals job name `ci` exactly; workflow run once before selecting it (Pitfall 1). Acceptance: the proof PR's check actually reports red (not stuck on "Expected"). |
| T-03-04 | Tampering | Deliberately-failing test leaks into a tracked branch | mitigate | Throwaway PR closed without merge; branch deleted; failing test confined to the throwaway branch. Acceptance: no failing test on tracked branches; branch cleaned. |
</threat_model>
