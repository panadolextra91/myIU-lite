<objective>
Deliver the login vertical slice end-to-end: a user authenticates with username+password, receives a role-carrying JWT in HttpOnly cookies, can read their identity via GET /auth/me, and can log out. This plan also lands all shared backend auth infrastructure (migration 000003, sqlc user queries, JWT helpers, CORS, the AuthMiddleware skeleton, config fields) and bootstraps the entire frontend stack (axios, react-router, Zustand, TanStack Query, shadcn/ui + Tailwind v4) with a working Login page and role landing pages.

Purpose: This is the auth spine every later route hangs off. Establishes the cookie-JWT transport (D-11/D-12), the session-invalidation primitive (password_changed_at, D-13), and the frontend shell. Demonstrable against the seeded bootstrap admin (admin/123456).

Output: Working /auth/login, /auth/me, /auth/logout endpoints; the AuthMiddleware chain (steps 1-4); a frontend that logs in and routes by role. AUTH-01 (login + role JWT) and AUTH-02 (logout) land here. The must_change_password allow-list (step 5) is wired in plan 02; RequireRole + refresh + guards in plan 03.

## Phase Goal

**As a** myIU user (student, lecturer, or admin), **I want to** log in with my username and password and be taken to my role's area, **so that** I can access the parts of the platform meant for my role without using email.
</objective>
