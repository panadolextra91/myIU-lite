<objective>
Deliver the RBAC + ownership + token-refresh vertical slice end-to-end: the backend RequireRole middleware rejects role mismatches with 403, POST /auth/refresh re-mints access tokens while honoring the password_changed_at kill-switch (so a changed/reset password also kills the long-lived refresh token), and the frontend transparently refreshes once on a 401 and gates routes with ProtectedRoute + RoleGuard inside the separate /student, /lecturer, /admin route trees (D-20/D-21).

Purpose: This closes AUTH-05 (role gate + ownership, never trusting client IDs) and completes the D-12 two-token model + D-22 transparent refresh. The role split is established at the routing boundary (D-20) as the pattern every later phase reuses. RoleGuard is UX-only; the backend RequireRole is the security bar (success criterion 4 — "bypassing the SPA does not unlock endpoints").

Output: RequireRole middleware + a role-guarded route group demonstrating the gate; POST /auth/refresh with the full chain; FE 401 interceptor + ProtectedRoute + RoleGuard + AppLayout + role trees. AUTH-05 lands here; AUTH-01's session continuity (refresh) is completed.

## Phase Goal

**As a** logged-in user, **I want to** stay signed in across short token expiries and be blocked from areas outside my role, **so that** I only ever reach the parts of myIU my role and ownership permit.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JWT role claim → RequireRole | Role is trusted only after AuthMiddleware verifies the token signature |
| Refresh cookie → /auth/refresh | The long-lived refresh token is untrusted until it passes the full chain incl. iat check |
| SPA RoleGuard → backend RequireRole | The SPA guard is UX; the server gate is the real authorization boundary |

| T-02-15 | Spoofing | /auth/refresh stale session | mitigate | Refresh re-runs load-user + iat>=password_changed_at + account-status before minting; refresh token honors the kill-switch (Pitfall 3, Open Item 2) — Task 1 |
