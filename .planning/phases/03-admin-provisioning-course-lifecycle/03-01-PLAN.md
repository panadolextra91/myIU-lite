|----------|-------------|
| Browser SPA → /admin/* API | Untrusted client; admin identity asserted only by the verified JWT cookie, never by client-supplied IDs |
| App code → audit_log table | Application must not be able to rewrite history; the DB itself is the enforcement point |
| Seed data → auth path | The SYSTEM account is data in the users table that must never become an authentication subject |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Repudiation | audit_log mutation (cover tracks) | mitigate | BEFORE UPDATE/BEFORE DELETE triggers RAISE EXCEPTION (D-35); integration test asserts UPDATE+DELETE fail (Task 1) |
| T-03-02 | Elevation of Privilege | SYSTEM account login | mitigate | Sentinel non-bcrypt password_hash '!' makes bcrypt.CompareHashAndPassword error; is_system excluded from listings; test asserts 401 (Task 2, D-38) |
| T-03-03 | Information Disclosure | audit metadata over-exposure | mitigate | Viewer is read-only and admin-only (RequireRole admin); only audit columns returned, no password material ever lands in audit metadata |
