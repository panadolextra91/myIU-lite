<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /admin/users* API | Untrusted CSV file + JSON; never trust client validation or client-supplied actor/role |
| Uploaded CSV → server memory | Untrusted file content/size crosses into the importer |
| Derived password → storage | Secret material; only the bcrypt hash may persist |

## STRIDE Threat Register (ASVS L1, block on high)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Denial of Service | CSV upload size | mitigate | http.MaxBytesReader + MaxMultipartMemory cap (~5MB), 413 on oversize (Task 2, Pitfall 6) |
| T-03-08 | Tampering | partial/inconsistent import on failure | mitigate | Two-phase validate-all-then-insert in one pgx.Tx; any error rolls back all (D-27, Task 1) |
| T-03-09 | Information Disclosure | plaintext/generated password leakage | mitigate | Never store/return generated password; only bcrypt hash persists; reset returns no password (D-26) |
| T-03-10 | Tampering | CSV-injection / SQL-injection via CSV fields | mitigate | sqlc parameterized inserts only; CSV fields are values, never concatenated into SQL; full_name/id stored as data |
| T-03-11 | Elevation of Privilege | non-admin hitting account endpoints | mitigate | All /admin/users* behind AuthMiddleware + RequireRole(admin) |
| T-03-12 | Spoofing | client-supplied actor_id on audit | mitigate | actor_id from c.Get("user_id") JWT claim only (RESEARCH A4) |
| T-03-13 | Elevation of Privilege | importing/listing the SYSTEM account | mitigate | GetActiveUsernames + ListUsers filter is_system=FALSE; manual create rejects reserved username |
| T-03-SC | Tampering | npm/shadcn install | mitigate | shadcn add copies source only; no new runtime deps (RESEARCH audit n/a) |
</threat_model>
