<objective>
Lay the schema foundation for Phase 5 and complete the one cross-phase touch the gradebook AUTO computation depends on: migration `000008` creates every Phase-5 table AND adds `max_score` + `grading_finalized_at` to the Phase-4 `assignments` table (D-57, D-64). Deliver the assignments touch as a working vertical slice — a lecturer can set a max score when creating an assignment and finalize grading on it — so that downstream gradebook AUTO normalization (Assignment Average) and eligibility have real columns to read.

Purpose: D-57 requires assignments to define a maximum score (Phase 4 has none); D-64 requires an explicit "grading finalized" marker for assignment eligibility. Both are prerequisites of the gradebook compute slice (05-02/05-03). Co-locating the full `000008` DDL here means downstream plans only add queries, never schema.
Output: Migration `000008` up/down, ALTER on assignments, `FinalizeAssignmentGrading` query + lecturer action, `max_score` carried through the assignment-create flow, regenerated sqlc, an integration test, and the FE form field for max score.
</objective>
