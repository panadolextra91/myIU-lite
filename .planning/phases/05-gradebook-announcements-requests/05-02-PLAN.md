| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Elevation | Lecturer creates/edits a scheme or enters scores for a course they don't own | mitigate | `AssertCourseMember(ctx, pool, courseID, lecturerID, db.UserRoleLecturer)` before every scheme/score mutation; derive lecturer from JWT, never client course_id |

| T-05-06 | Tampering | Out-of-range MANUAL score (negative / >100) via API or CSV | mitigate | DB CHECK (0<=score<=100) + service clamp/validate + CSV per-row 0..100 check |
| T-05-07 | DoS | CSV bomb / oversized upload | mitigate | `http.MaxBytesReader(c.Writer, c.Request.Body, 5<<20)` + extension/content-type check (reuse enrollments) |
| T-05-08 | Tampering | Spreadsheet-formula injection in CSV cells | mitigate | Reuse `strings.TrimLeft(v, "=+-@\t\r ")` sanitization from enrollments ParseCSV |
| T-05-09 | Info Disclosure | Lecturer A computes grades for course B's students | mitigate | All compute reads scoped by course_id after AssertCourseMember; student roster via ListCourseStudents for that course only |
