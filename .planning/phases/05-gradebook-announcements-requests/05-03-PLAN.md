<objective>
Complete the gradebook vertical slice: a lecturer publishes a top-level component (freezing each student's computed value into a snapshot and notifying every affected student in the same transaction — D-59/D-66/GRADE-05/NOTIF-02), and a student views their grades reading the frozen snapshot (D-66), with the overall appearing only once all required top-level components are published. This is what makes grades student-visible and auto-notified.

Purpose: GRADE-05 (student views grades; availability auto-notified) and GRADE-04 completion (student overall over snapshots). Builds directly on the scheme + live computation from 05-02. The new logic is the published-snapshot persistence model and the snapshot-vs-live split.
Output: publication queries, `PublishComponent`/`RepublishComponent` service (same-tx fan-out notify), student snapshot reads, the student Grades page, route wiring, and an anti-theater integration test for atomicity + frozen-vs-live + republish.
</objective>

| T-05-10 | Info Disclosure | Student reads another student's grades (IDOR) | mitigate | `GetStudentGrades` scopes the read to `student_id = c.GetInt64("user_id")`; never accept a client-supplied student_id |
