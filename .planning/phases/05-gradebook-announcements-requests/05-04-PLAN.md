<objective>
Deliver announcements as a complete vertical slice: a lecturer composes an announcement targeted to all or specific enrolled students (D-60), and each targeted student receives a persisted notification + can browse the per-course Announcements page (ANNC-01/02/03). Announcements are first-class immutable entities (D-60/D-61) and delivery reuses the Phase-4 notification primitive verbatim.

Purpose: ANNC-01 (all students), ANNC-02 (specific students), ANNC-03 (persisted, no email). The notification system is reused without becoming the source of truth — "announcements are content; notifications are delivery."
Output: `announcements.sql`, the `internal/announcements/` feature folder, lecturer composer + history page, student browse page, route + main.go wiring, and an anti-theater fan-out test (ALL/SPECIFIC atomicity + recipient correctness).
</objective>
