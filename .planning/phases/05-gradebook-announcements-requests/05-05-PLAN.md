<objective>
Deliver student↔lecturer requests as a complete vertical slice: a student sends a directed typed request to a chosen course lecturer (D-62), the lecturer replies once with a decision + optional note (D-63), and both directions auto-deliver notifications in-transaction (REQ-01/02/03, NOTIF-02). This closes the phase goal — a course can be run to completion without email.

Purpose: REQ-01 (typed directed request), REQ-02 (yes/no reply), REQ-03 (reply auto-delivered). Directed routing eliminates shared-inbox races — "requests have a clear owner; directed, not broadcast." Symmetric: request-creation notifies the lecturer, reply notifies the student.
Output: `requests.sql`, the `internal/requests/` feature folder, student compose page, lecturer inbox + reply dialog, route + main.go wiring, and an anti-theater test (directed visibility + both-direction notify atomicity + closed-permanently).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client (student SPA) → /api/student | Untrusted course_id + targetedLecturerID cross here |
| client (lecturer SPA) → /api/lecturer | Untrusted request_id for reply; reply scoped to JWT user_id |
| API → Postgres | Same-tx request mutation + notification (both directions) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-18 | Info Disclosure | A lecturer other than the targeted one reads/replies to a request | mitigate | Every lecturer read filtered by `targeted_lecturer_id = c.GetInt64("user_id")`; ReplyRequest UPDATE guarded by `targeted_lecturer_id=$2`; ErrNotTargeted otherwise (D-62) |
| T-05-19 | Spoofing/Elevation | Student targets a lecturer not assigned to the course | mitigate | Validate targetedLecturerID against `ListCourseLecturers` of that course before insert |
| T-05-20 | Info Disclosure | Student reads another student's requests | mitigate | `ListForStudent`/`GetByID` scoped to `student_id = JWT user_id` |
| T-05-21 | Tampering | Re-reply / reopen a closed request | mitigate | `ReplyRequest` UPDATE guarded by `status='PENDING'`; second reply is a no-op → ErrAlreadyClosed (D-63) |
| T-05-22 | Integrity | Reply status changes without the student notification (or vice versa) | mitigate | Single `pool.Begin`/`Commit`; test asserts neither-on-failure (NOTIF-02) |
| T-05-SC | Tampering | npm/go package installs | accept | No new packages this phase (RESEARCH audit: none) |
